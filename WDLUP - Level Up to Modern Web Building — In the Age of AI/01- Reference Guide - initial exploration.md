# Website Definition Reference Guide

Use this guide before starting any web project — for yourself or for a client.
It helps surface the questions that are easy to skip but that shape everything that follows.

You don't need to answer all of this in detail. Just enough to give the project a clear direction.

## 9. Easy-to-overlook questions

These are the ones that often get skipped — but that affect the outcome significantly.

- **Mobile first?** Most visitors will be on mobile? If so, we'll need to indicate it to design primarily for that.
- **Language / locale?** One language or more?
- **Accessibility baseline?** At minimum: readable contrast, keyboard-navigable, no tiny text. But we might have some strict requirements around this
- **Does it need to rank on search (SEO)?** If yes, page titles and structure matter from day one.
- **How will people find this site?** (This affects what the homepage needs to say)
- **What does success look like in 3 months?** (Even a rough answer helps set the right scope)

---

> **Remember:** this guide is just a tool, not a checklist to complete perfectly.
> Answer what you can. Leave blanks where you're unsure. Revisit as the project evolves.
